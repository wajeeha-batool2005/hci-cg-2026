Norman vs. GOMS vs. KLM: Key Differences
Aspect Norman GOMS KLM
Category Cognitive / descriptive model Predictive / analytic model Predictive / quantitative model
Explains User's interaction with system, errors, feedback, gulfs Task structure for expert users Low-level physical and motor actions
Predicts Nothing (no performance prediction) Relative efficiency Execution time
Structure of cycle Linear cycle Hierarchical recursion Linear execution
Phases 2 phases:
  • Execution
  • Evaluation
 Hierarchical components, no linear phases Execution phase (ignore acquisition phase)
Stages 7 stages:
  • Goal
  • Plan/ Intention
  • Specify action
  • Execute action
  • Perceive
  • Interpret
  • Evaluate
 4 hierarchical elements:
  • Goal
  • Methods
  • Selection rules
  • Operators
 7 primitive operators:
  • [K] - Keystroke
  • [B] - Button press
  • [P] - Pointing
  • [H] - Homing
  • [D] - Drawing
  • [M] - Mental preparation
  • [R] - System response
Processes Thinking → Action → Evaluation + Gulfs Planning → Actions Actions + Time
Limitations No time prediction No system response, no error (expert user) No cognition, no error (expert user)
Steps of Interaction
  1. Form goal (what to achieve)
  2. Plan/ intent ways to achieve goal
  3. Specify the sequence of action
  4. Execute action
  5. Perceive system state (feedback by system)
  6. Interpret the response of system
  7. Evaluate outcome with respect to goal defined
  1. Define goal
  2. Break/ decompose goal into subgoals/ methods
  3. Apply selection rules to choose method from alternatives
  4. Break method into operators and execute
  5. Return to higher goal or complete task (hierarchical task analysis)
  1. List task's physical actions/keystroke-level operators
  2. Insert mental operators (M) using heuristics/rules
  3. Sum predefined times for each operator
  4. Predict total execution time

Key Takeaway

Norman’s model focuses on the user’s cognitive cycle of goals and evaluation, GOMS structures user behaviour into goals, methods, and operators, while KLM removes cognition details and quantifies interaction efficiency in time.
