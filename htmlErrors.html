HTML Error Identification & Correction

This page demonstrates common HTML structure and accessibility mistakes, along with their corrected versions, within the scope of Lab 01.

Common HTML errors with corrected solutions
UI Element / Issue Wrong HTML (with mistake) Correct UI Element Correct HTML
Anchor directly inside ul (missing li) 
<nav>
  <ul>
    <a href="index.html">Home</a>
    <li><a href="cv.html">CV</a></li>
  </ul>
</nav>
 Navigation list where each link is inside a list item 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="cv.html">CV</a></li>
  </ul>
</nav>
Multiple h1 elements 
<h1>Page Title</h1>
<h1>Section Title</h1>
 Single main heading 
<h1>Page Title</h1>
<h2>Section Title</h2>
Heading hierarchy is incorrect 
<h1>Analysis</h1>
<h3>Norman Model</h3>
 Correct heading hierarchy 
<h1>Analysis</h1>
<h2>Norman Model</h2>
Image missing alt attribute 
<img src="norman.png">
 Image with alt text 
<img src="norman.png" alt="Norman action cycle diagram">
Form label not associated with its control (for ≠ id) 
<label for="msg">Message</label>
<textarea id="message" name="message"></textarea>
 Label correctly connected to form control 
<label for="message">Message</label>
<textarea id="message" name="message"></textarea>

Multiple aria-current values 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html" aria-current="page">Home</a></li>
    <li><a href="cv.html" aria-current="page">CV</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>
 Only one current page 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="cv.html" aria-current="page">CV</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

True / False

  • Every link inside <ul> must be placed inside an <li>.
    ✅ True.
  • A page may have multiple h1 headings to make text larger.
    ❌ False.
  • aria-current="page" should be used on the active navigation item.
    ✅ True.
  • A form is accessible even if it has no <label> elements.
    ❌ False.
  • KLM predicts execution time for expert, error-free performance.
    ✅ True.

Short Answers

  • Semantic vs generic elements: Semantic elements convey meaning and structure (e.g., nav), while generic elements like div have no inherent meaning.
  • KLM limitation: KLM focuses on low-level physical actions (keystrokes, pointing, clicking etc.) and does not model planning, interpretation, or decision-making.
