Introduction to HCI Models:

Human–Computer Interaction (HCI) models are theoretical frameworks that explain how users interact with computer systems. These models focus on cognitive processes, task performance, and system feedback to create effective and user-friendly interfaces. HCI models guide interface design, evaluation, and optimization.

Key HCI Models:

Norman’s Model

Norman's model of interaction describes the process of interaction between a human user and a computer system. The model highlights the cognitive steps involved in interacting with a computer and helps identify usability issues.

Execution-Evaluation Cycle:

The Norman's interaction cycle can be divided into two major phases:

  • Execution phase - user plans in this phase and perform actions on the system to achieve the goal.
  • Evaluation phase - in this phase user observe and evaluate the system's response with respect to the desired goal.

Gulfs:

Gulfs refers to the gaps between the user's mental model and the system's actual capabilities or feedback. Norman's model highlights two gulfs:

  • Gulf of execution - it represents the effort needed by users to convert their objectives into actions that the system permits.
  • Gulf of evaluation - it represents the gap between system's response and the user's expectation.

7 Stages of Interaction Cycle:

7 Stages of Norman's Execution-Evaluation Model (Dix et al., Chapter 3 pg#126)
Phase Stage Example
Execution Establishing the goal It's getting dark, need more light in room
Forming the intention Turn on the desk lamp to increase the light
Specifying the action The user decides to reach out and press the lamp switch
Executing the action The user physically presses the switch on the desk lamp
Evaluation Perceiving the system state The user observes whether the light turns on or remains off
Interpreting the system state If the light does not turn on, the user interprets this as a possible fault, such as a blown bulb or unplugged lamp
Evaluating the system state The user evaluates whether the new lighting condition satisfies the original goal of having enough light

GOMS Model

GOMS of Card, Moran, and Newell breaks down skilled user tasks into four key elements—Goals, Operators, Methods, and Selection rules—to predict performance in human-computer interaction. It focuses on planning and action stages while excluding system evaluation, as it assumes expert users with no need for feedback interpretation.

Goals

Goals represent what users aim to accomplish. They are structured hierarchically as "memory points" for decision-making and error recovery.

Operators

These are the basic, atomic actions at the lowest analysis level, including physical moves (e.g., press 'X' key, move mouse) or mental ones (e.g., read dialog). GOMS model consist of 7 operators, used to define the actions performed by user.

  • K - keystroke/ keypress
  • B - button press
  • P - pointing
  • H - homing
  • D - drawing
  • M - mental preparation
  • R - system response

Methods

Methods are alternative sequences of subgoals and operators to fulfill a goal. They are organized in a hierarchical structure, where a high-level goal is decomposed into lower-level actions.

Selection Rules

Selection rules in the GOMS model serve as decision-making mechanisms that systematically determine which method to apply when multiple methods exist for achieving a single goal.

KLM (Keystroke-Level Model)

The Keystroke-Level Model (KLM) predicts user performance time for simple, routine unit tasks (under 20 seconds) by decomposing the execution phase of interaction into a small set of primitive motor operators. It assumes the user is an expert and the method has already been chosen. KLM is a low-level instance of the GOMS framework with only seven operators because it models execution, not thinking about what to do.

Two-Phase Task Structure

  • Acquisition Phase - user constructs mental representation of the task and plans method using system primitives. KLM ignores this completely - assumes expert user with method pre-decided.
  • Execution Phase - pure motor actions performed automatically by expert users, with no high-level cognition. KLM made predictions only on this stage.
